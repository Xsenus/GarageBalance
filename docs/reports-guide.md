# Отчёты GarageBalance

## Доступ

Раздел доступен пользователям с правом чтения отчётов. Backend проверяет permission для экранных данных и для каждого XLSX/PDF-экспорта.

## Общий принцип

Каждая вкладка отчётов содержит:

- период и дополнительные фильтры;
- итоговые показатели;
- серверную сортировку и пагинацию;
- таблицу с состояниями загрузки, ошибки и отсутствия данных;
- экспорт XLSX и PDF с теми же фильтрами.

Фильтрация, сортировка и агрегирование выполняются в PostgreSQL. Экран получает только выбранную страницу и полные итоги, поэтому объём истории не должен замедлять браузер.

## Вкладки

- Консолидированный отчёт — отдельный блок для каждого месяца: параллельные списки поступлений и выплат по наименованиям, их итоги и разница, а также фактический остаток банковского счёта на начало и конец месяца. Остаток счёта складывается из активных сдач кассы в банк за вычетом банковских выплат; выплаты без чека из кассы его не уменьшают.
- По гаражам — услуги, начисления, поступления и разница по выбранным гаражам. Гаражи выбираются так же, как в форме платежей: поиск по номеру или владельцу, флажки в результатах, карточки выбранных записей, удаление одной карточки и общая очистка. В подробном виде услуга показана перед начислением. Пояснение над таблицей фиксирует формулу разницы и предупреждает, что группировка объединяет услуги в одну строку по гаражу и месяцу.
- По выплатам — поставщики, сотрудники, начисления и выплаты. Поставщиков и сотрудников можно найти в общем поле и отметить несколькими флажками; выбранные записи показаны карточками и снимаются по одной или общей очисткой. В общем режиме начисление и выплата одного месяца по одной услуге и одному контрагенту показаны в общей строке; отдельные режимы сохраняют детализацию операций.
- Поступления — операции кассы и оплаты владельцев. Гаражи выбираются поиском по номеру или владельцу и несколькими флажками с теми же карточками выбранных записей.
- Оплаты из кассы — расходные операции.
- Сдача кассы в банк — банковские передачи.
- Сборы — начислено, собрано и должники по вариации сбора.
- Изменение фондов — пополнения, изъятия и остатки.

Финансовый отчёт из карточки гаража, поставщика или сотрудника по умолчанию открывается за весь период обслуживания в системе: от первой активной финансовой записи до текущего месяца либо до последнего будущего месяца, если он уже есть. Для записи без операций выбирается текущий месяц. Отменённые строки не расширяют период, а даты после открытия можно изменить вручную.

Во время первоначальной загрузки окно сохраняет высоту готового отчёта и показывает каркас итогов и таблицы. Календарь периода можно раскрыть сразу: он не обрезается из-за временно пустого содержимого, а изменение размеров окна после получения данных сведено к минимуму.

Отчёт поставщика начинает расчёт с входящего остатка на первый месяц выбранного периода. В него входят стартовый баланс и вся активная история начислений и выплат до границы периода; отменённые операции не учитываются. Входящий остаток показывается отдельно и не подменяет сумму начислений выбранного периода.

## Сортировка и пагинация

Сортируемое поле передаётся в API как разрешённый ключ. Backend не принимает произвольное имя SQL-колонки. Повторное нажатие меняет направление, сброс возвращает исходный порядок.

Страница содержит 10, 25, 50 или 100 строк. Изменение фильтра, сортировки или размера страницы возвращает пользователя на первую страницу.

## XLSX и PDF

Экспорт формируется POST-запросом, чтобы фильтры безопасно передавались структурированным DTO. Файл содержит всю подходящую выборку, а не только видимую страницу. Имя файла включает тип отчёта и период.

Если в фильтре с флажками ничего не отмечено, отчёт и его экспорт формируются по всем доступным записям. Отмеченные значения одинаково применяются к экранной таблице, XLSX и PDF.

В отчёте по гаражам экран, XLSX и PDF используют один состав: `Месяц`, `Гараж`, в подробном режиме `Услуга`, затем `Начисления`, `Поступления` и `Разница`. Колонки владельца в выгрузке нет. Месяц показан как `мм.гггг`, после данных следуют итоговая строка и то же пояснение расчёта, что над экранной таблицей. В сгруппированном режиме услуга отсутствует во всех трёх представлениях.

Успешное формирование отчёта фиксируется в истории изменений. Ошибка скачивания показывается внутри вкладки и не блокирует другие разделы.

## Добавление нового отчёта

1. Зафиксировать назначение, источники, период, permissions и формулы итогов.
2. Реализовать bounded PostgreSQL query с whitelist сортировки.
3. Добавить DTO экранной страницы и отдельный экспортный запрос.
4. Добавить вкладку с общими фильтрами, таблицей и пагинацией.
5. Проверить пустой период, большие объёмы, границы дат и запрещённый доступ.
6. Сравнить экранные итоги с XLSX/PDF.
7. Добавить backend, PostgreSQL и frontend-тесты.
