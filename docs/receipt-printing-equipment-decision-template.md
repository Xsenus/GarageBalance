# Receipt Printing Equipment Decision Template

Этот шаблон используется после или вместе с выбором сценария печати. Он нужен, чтобы зафиксировать оборудование, способ подключения и условия приемки до разработки фактического адаптера.

## Source Context

- `docs/source-analysis.md`: для фискального сценария ориентиром названы АТОЛ 30Ф/30Ф+, для мобильного варианта - Эвотор 5i.
- `docs/receipt-printing-design.md`: фактическое устройство, PDF renderer или локальный print-сервис должны подключаться только через `IReceiptPrintingAdapter`.
- `docs/receipt-printing-scenario-decision-template.md`: сначала нужно понять, нужен фискальный чек, внутренняя квитанция или смешанный вариант.

## Roadmap Result

- Если выбран PDF/browser/local printer без фискализации, перевести в работу внутреннюю квитанцию.
- Если выбрано фискальное устройство или сервис, перевести в работу фискальную интеграцию и тесты выбранного сценария.
- Если решение отложено, оставить пункты реализации `[ ]`, а acceptance на устройстве - `[acceptance]`.

## Safe Data Rules

- Не переносить в Git IP/логины/пароли кассы, токены облака, fiscal storage identifiers, registration numbers, raw device logs, QR payloads с реальными данными, телефоны, адреса, банковские реквизиты или персональные документы.
- Для диагностики фиксировать только безопасный статус, короткий error code, время, correlation id и обезличенный сценарий.
- Секреты оборудования хранить только в environment/deployment secrets, user secrets, protected settings или локальных приватных файлах вне репозитория.

## Close Conditions

Пункт `Выбрать оборудование: ориентир АТОЛ 30Ф/30Ф+ для фискального сценария, Эвотор 5i для мобильного варианта` можно закрыть как `[x]` только после заполненного выбранного варианта, ответственного лица и способа подключения. До этого фактический адаптер конкретного устройства, тесты выбранного сценария и приемка на устройстве должны оставаться незакрытыми.
