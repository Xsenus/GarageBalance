# Receipt Printing Requisites Decision Template

Этот шаблон фиксирует обязательные реквизиты печатной квитанции или чека перед реализацией выбранного сценария. Он нужен, чтобы форма печати не была собрана по догадкам и не раскрывала лишние персональные или финансовые данные.

## Source Context

- `docs/project-roadmap.md`: Stage 10 требует уточнить обязательные реквизиты печатной квитанции/чека для внутреннего учета.
- `docs/receipt-printing-design.md`: минимальная внутренняя квитанция уже описана как проектная основа, но обязательность каждого поля требует решения заказчика.
- `docs/receipt-printing-scenario-decision-template.md`: юридический сценарий выбирается отдельно.
- `docs/receipt-printing-obligation-decision-template.md`: ответственное лицо и перечень операций выбираются отдельно.

- [ ] Номер гаража.

- [ ] Вид поступления.

## Roadmap Result

- После утверждения внутренних реквизитов можно переводить в работу `Реализовать печать внутренней квитанции`.
- После утверждения фискальных реквизитов и оборудования можно переводить в работу фискальную интеграцию.
- Если часть реквизитов запрещена, это нужно закрепить в backend/frontend тестах выбранного сценария.
- Если реквизиты не утверждены, реализация и тесты выбранного сценария остаются незакрытыми.

## Safe Data Rules

- Не переносить в Git реальные паспортные данные, полные адреса, телефоны, банковские реквизиты, fiscal storage identifiers, QR payloads с реальными данными, registration numbers кассы, токены и raw device logs.
- В документации использовать только названия полей, обезличенные примеры и placeholders.
- Если нужен пример квитанции, использовать тестового владельца, тестовый гараж и вымышленные реквизиты.

## Close Conditions

Пункт `Уточнить обязательные реквизиты печатной квитанции/чека для внутреннего учета` можно закрыть как `[x]` только после заполненного списка обязательных реквизитов, правил минимизации данных и решения по формату копии. До этого реализация выбранного сценария и тесты выбранного сценария должны оставаться незакрытыми.
