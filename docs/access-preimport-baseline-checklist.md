# Access Pre-Import Baseline Checklist

Этот checklist фиксирует безопасный baseline до фактического переноса старой Access-БД.

## Предусловия

- [ ] Есть приватная рабочая копия `.accdb`/`.mdb`; оригинал не изменяется.
- [ ] Есть рабочий reader или согласованная конвертация.
- [ ] Снят schema inventory: пользовательские таблицы, колонки, ключи, связи, индексы и row counts.
- [ ] Baseline готовится вне Git и без raw rows.

## Что фиксировать до импорта

- [ ] SHA-256 и размер рабочей копии Access-БД.
- [ ] Дата и время baseline-среза.
- [ ] Provider/способ чтения.
- [ ] Row counts по ключевым таблицам: гаражи, владельцы, платежи/поступления, выплаты, счетчики, тарифы/виды оплат, поставщики.
- [ ] Checksum или агрегаты по каждой ключевой таблице без персональных строк.
- [ ] Список таблиц, исключенных из импорта, и причина исключения.
- [ ] Список предупреждений: пустые таблицы, неожиданные row counts, поврежденные/недоступные таблицы.

## Безопасные агрегаты

Разрешено сохранять:

- имя таблицы;
- количество строк;
- технический checksum;
- min/max технического идентификатора;
- количество пустых обязательных полей;
- количество дублей по техническому ключу.

Запрещено сохранять:

- ФИО, адреса, телефоны, паспортные данные;
- номера документов и платежные строки по конкретным владельцам;
- raw rows или screenshots с таблицами;
- сам `.accdb/.mdb`, exports, dumps и private import folders.

## Пример результата

```text
copySha256=<hash>
provider=Microsoft.ACE.OLEDB.16.0
baselineAt=YYYY-MM-DDTHH:mm:ss
rawRowsExported=false
tables:
  Garages: rows=1234 checksum=<hash>
  Owners: rows=987 checksum=<hash>
  Payments: rows=54321 checksum=<hash>
```

## Когда roadmap-пункт можно закрыть

- [ ] Baseline снят с приватной рабочей копии.
- [ ] Row counts и checksums есть для всех ключевых таблиц.
- [ ] Итог не содержит персональных/финансовых строк.
- [ ] Privacy-check подтверждает, что Access-файлы и raw exports не попадут в Git.
- [ ] Baseline сохранен так, чтобы после импорта можно было сверить количество перенесенных объектов и проблемных строк.
