# Расширенная Приемка Отчетов До Одного Месяца

Этот документ фиксирует договоренность из переписки: замечания по отчетам могут приниматься в расширенном окне до одного месяца, потому что заказчику нужен полный рабочий месячный цикл для проверки начислений, поступлений, выплат и итоговых отчетов.

## Что считается учтенным в проекте

- `docs/source-analysis.md` хранит исходную договоренность: для замечаний по отчетам допускается расширенная приемка до одного месяца без изменения договора.
- `docs/monthly-cycle-checklist.md` включает это правило в закрытие месяца и напоминает сохранять выгрузки XLSX/PDF вне Git.
- Roadmap больше не смешивает само правило с фактической приемкой: пункт Stage 7 отмечает, что правило учтено, а ручная проверка отчетов на реальном месячном цикле остается отдельным acceptance-условием.

## Что остается ручной приемкой

- Заказчик или пользователь должен проверить отчеты на реальном или согласованном тестовом месячном цикле.
- Замечания по отчетам нужно добавить в roadmap отдельными задачами или подтвердить, что мотивированных замечаний нет.
- Если замечания связаны с реальными персональными, платежными или импортными данными, в Git переносится только обезличенное описание проблемы и шаги воспроизведения.

## Проверяемые ссылки

- `MonthlyCycleChecklistTests.MonthlyCycleChecklistCoversRequiredOperationalSteps` закрепляет наличие правила в ежемесячном регламенте.
- `StageSevenReportExtendedAcceptanceRuleIsMarkedCompleteWhenSourceMonthlyChecklistAndRoadmapSeparateManualAcceptance` закрепляет закрытый статус правила в roadmap и отдельный остаток ручной приемки.
