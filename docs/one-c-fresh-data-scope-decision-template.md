# 1C Fresh Data Scope Decision Template

Этот шаблон нужен до фактического обмена с 1C Fresh. Он фиксирует, какие справочники и документы синхронизируются, в каком направлении и с какими ограничениями. Подключение, токены и реальные данные в этот файл не записываются.

## Source Context

- `docs/project-roadmap.md`: Stage 9 оставляет параметры API, направление обмена и состав документов в `[decision]`.
- `docs/one-c-fresh-sync-design.md`: модель синхронизации уже предусматривает preview, retry, conflict handling, journal и запрет plaintext secrets.
- `docs/one-c-fresh-contour-acceptance-checklist.md`: приемка на контуре требует подтвержденных параметров API, направления обмена и состава документов.

## Direction

Выбрать один вариант:

- [ ] Только выгрузка из GarageBalance в 1C Fresh.
- [ ] Только загрузка из 1C Fresh в GarageBalance.
- [ ] Двусторонняя синхронизация.
- [ ] Только ручной preview без apply до отдельного решения.
- [ ] Решение отложено.

Подтвердить:

- [ ] Какая система является источником истины для владельцев/контрагентов.
- [ ] Какая система является источником истины для платежей.
- [ ] Какая система является источником истины для начислений.
- [ ] Какая система является источником истины для возвратов, корректировок и отмен.

## Dictionaries

Отметить, что входит в обмен.

- [ ] Владельцы/контрагенты.
- [ ] Гаражи/объекты учета.
- [ ] Поставщики.
- [ ] Виды поступлений.
- [ ] Виды выплат.
- [ ] Тарифы/услуги.
- [ ] Прочие справочники:

Для каждого выбранного справочника указать безопасно:

- ключ сопоставления: внешний id, ИНН, номер гаража, код, name+period или другой ключ;
- кто создает новую запись при отсутствии пары;
- что делать с архивными/удаленными записями;
- какие поля нельзя передавать из-за персональных данных.

## Documents And Operations

Отметить, что входит в обмен.

- [ ] Платежи членов ГСК.
- [ ] Начисления по тарифам.
- [ ] Начисления поставщиков.
- [ ] Выплаты/расходы.
- [ ] Показания счетчиков.
- [ ] Счета, чеки, квитанции.
- [ ] Акты или бухгалтерские документы 1C Fresh.
- [ ] Отчеты или агрегаты за период.

Для каждого выбранного документа указать:

- период обмена;
- правила округления сумм;
- правила отмен/возвратов/корректировок;
- idempotency key;
- conflict rule: skip, update, create correction, manual review;
- audit rule в GarageBalance.

## Preview And Apply Gates

- [ ] Preview обязателен перед apply.
- [ ] Apply разрешен только после подтвержденного preview без блокирующих конфликтов.
- [ ] Conflicts нельзя применять автоматически без выбранного правила.
- [ ] Retry создает новый запуск или явно связанную попытку.
- [ ] Результат обмена записывается в audit/journal без plaintext secrets и без персональных/финансовых строк сверх необходимого минимума.

## Safe Data Rules

Не записывать в этот файл и не коммитить:

- токены 1C Fresh, refresh token, client secret, cookie, JWT, connection string;
- реальные телефоны, адреса, паспортные данные, банковские реквизиты;
- полные выгрузки платежей, начислений, контрагентов и документов;
- raw API responses с персональными, финансовыми или секретными значениями;
- дампы БД, backup-файлы, `.accdb`, `.mdb` и приватные import folders.

- Состав справочников подтвержден: да/нет
- Состав документов подтвержден: да/нет
- Правила конфликтов подтверждены: да/нет
- Preview/apply gates подтверждены: да/нет
- Что перенести в roadmap после решения:

```
Кратко записать только безопасный итог: какие сущности синхронизируются, направление, какие правила конфликтов нужны и какие вопросы остаются.
```

## Close Conditions

Открытый вопрос `Нужно подтвердить, какие именно данные 1C Fresh должны синхронизироваться` можно закрыть как `[x]` только после заполненного decision record, подтвержденного направления обмена, состава справочников/документов, правил конфликтов и безопасной проверки preview/apply-gates без секретов и реальных персональных/финансовых строк в Git.
