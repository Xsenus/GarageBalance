# Руководство пользователя

Инструкция для ежедневной работы в GarageBalance. Документ описывает рабочие разделы интерфейса: вход, справочники, тарифы, платежи, отчеты, импорт, историю изменений и "Что нового".

## 1. Вход

- [ ] Открыть локальный адрес `http://127.0.0.1:5173` или стенд `https://sgk.blagodaty.ru`.
- [ ] Ввести email и пароль.
- [ ] Если система запускается впервые, создать первого администратора через стартовую форму.
- [ ] Пароль должен быть не короче 8 символов.
- [ ] Если форма показывает validation summary, исправить отмеченные поля до повторной отправки.
- [ ] Если вход временно ограничен после нескольких ошибок, дождаться окончания ограничения или обратиться к администратору.
- [ ] После истечения срока сессии система автоматически очистит авторизацию и вернет форму входа; войти повторно, чтобы продолжить работу.

## 2. Права и видимость разделов

Разделы появляются по правам пользователя:

- Недоступные разделы полностью скрываются из главного меню и боковой панели; перейти в них по внутренней ссылке также нельзя.

- `users.manage` открывает "Пользователи".
- `dictionaries.read` открывает "Справочники" и "Тарифы и сборы".
- `tariffs.manage` разрешает изменять данные в разделе "Тарифы и сборы".
- `payments.read` открывает "Платежи".
- `reports.read` открывает "Отчеты", но для фильтров отчеты также требуют `dictionaries.read`.
- `import.run` открывает "Импорт".
- `audit.read` открывает "История изменений".
- `app_releases.manage` показывает права для "Что нового".

Если права нет, интерфейс показывает состояние "раздел недоступен" и не загружает защищенные данные этого раздела.

## 3. Пользователи и безопасность аккаунта

Раздел "Пользователи" доступен администратору.

- [ ] Создать сотрудника: email, имя, роль и пароль.
- [ ] Проверить роль по матрице прав перед сохранением.
- [ ] Отключить сотрудника при увольнении или смене доступа.
- [ ] Обычные изменения пользователя и прав роли сохраняются сразу; отдельное подтверждение остаётся при отключении, удалении и восстановлении доступа.
- [ ] Не отключать последнего активного администратора.
- [ ] Для собственного пароля открыть "Безопасность аккаунта", ввести текущий пароль, новый пароль и повтор.
- [ ] В разделе "Настройки" выбрать внутреннюю вкладку "Безопасность"; остальные группы параметров будут добавляться в эту боковую навигацию отдельно.
- [ ] Администратор может открыть вкладку "Отображение" и включить общую ведомость платежей при входе в раздел; стандартно этот режим выключен.
- [ ] Только системный администратор видит вкладку `Рабочая дата`. Она используется для тестирования начислений, просрочки и должников; после проверки администратор обязан вернуть автоматическую системную дату. Подробный сценарий приведён в `docs/admin-operations-guide.md`.
- [ ] Интеграции 1С Fresh и печати временно не показываются ни одной роли. Администратор может открыть `Настройки` → `Интеграции`, безопасно сохранить API-ключ DaData и использовать подсказки адресов в карточках гаражей и поставщиков; сохранённый ключ обратно не отображается.
- [ ] Не передавать пароль в открытых сообщениях.

## 4. Справочники

Раздел "Справочники" используется для базовых данных учета.

- [ ] Владельцы: ФИО, телефон, email и примечание.
- [ ] Телефоны владельцев, поставщиков и контактов вводить в российском формате `+7 (999) 123-45-67`. Допускается начать ввод с `8`, `+7` или с десяти цифр без кода страны: форма расставит скобки и дефисы автоматически. Пустое поле допустимо, неполный номер сохранить нельзя.
- [ ] Гаражи: номер, владелец, площадь, количество людей, стартовый баланс и стартовые показания счетчиков.
- [ ] При вводе адреса выбрать подходящий вариант DaData. После выбора адрес останется в поле без дополнительной успешной подписи; сообщения поиска, отсутствия вариантов и недоступности сервиса остаются видимыми.
- [ ] На вкладке `Контрагенты` → `Гаражи` использовать фильтр по части номера и диапазоны количества людей/этажей; кнопка `Сбросить фильтры` очищает все три условия, а режим `Показать должников` пересекается с ними.
- [ ] На вкладке `Контрагенты` → `Поставщики` отображается общий список без переключателя должников. Для проверки задолженности отсортировать одноимённую колонку.
- [ ] Финансовый отчёт открывать из карточки или таблицы уже сохранённого гаража, поставщика либо сотрудника. При первом открытии система сама выбирает весь период от первой активной финансовой записи до текущего или последнего будущего месяца; даты затем можно изменить вручную. В форме новой записи этой кнопки нет: сначала сохранить карточку, затем открыть её повторно.
- [ ] В разделе `Отчёты` → `По гаражам` найти нужные гаражи по номеру или ФИО владельца и отметить их флажками. Выбранный гараж можно убрать отдельной кнопкой, а весь фильтр — действием `Очистить`; если карточек нет, отчёт охватывает все гаражи. Подробная таблица показывает услугу перед начислением. Разница считается как начисления минус поступления; при группировке услуги объединяются в одну строку по гаражу и месяцу.
- [ ] В форме сотрудника выбрать отдел из списка. Если нужно перейти к ФИО, ставке или другому полю без изменения отдела, достаточно кликнуть вне списка: он закроется, а прежний выбор сохранится.
- [ ] В таблице отделов рабочими колонками являются `Отдел` и `Действия`. Отдельного статуса `Активен` нет; архивный отдел выделяется приглушённой строкой и предлагает действие восстановления.
- [ ] Группы поставщиков: группировка поставщиков по услугам.
- [ ] Поставщики: название, ИНН, группа, стартовый баланс и примечание; стартовый баланс доступен и при создании, и при изменении основной карточки в разделе `Контрагенты`.
- [ ] Телефон, почту, ФИО и должность поставщика изменять в таблице `Контакты` его основной карточки. Первый действующий контакт используется в общем списке поставщиков; отдельных конкурирующих полей телефона и почты внизу карточки нет.
- [ ] Виды поступлений: членские, целевые, вступительные и другие поступления.
- [ ] Статьи расходов: электроэнергия, мусор, вода, банковские и прочие расходы.
- [ ] Архивирование скрывает запись из рабочих списков, но не удаляет историю и audit.
- [ ] Перед архивированием система показывает подтверждение.
- [ ] Если запись архивирована ошибочно, включить "Показывать архивные", открыть действие "Вернуть" и подтвердить восстановление.
- [ ] Если восстановление показывает конфликт, сначала проверить активную запись с тем же номером, названием или кодом и исправить дубль.
- [ ] Изменения основной карточки поставщика сохраняются сразу по кнопке `Сохранить`, без второго окна со сравнением полей. Удаление или восстановление контакта по-прежнему требует отдельного подтверждения, а для удаления необходимо указать причину.

## 5. Тарифы и сборы

Раздел "Тарифы и сборы" доступен пользователям с `dictionaries.read`; для изменения данных требуется `tariffs.manage`.

- [ ] При добавлении услуги включить "Регулярные платежи" только для услуги, которая начисляется по расписанию. Если переключатель выключен, указать стоимость: после сохранения услуга появится в таблице "Нерегулярные платежи" без признаков счетчика и пороговой тарификации.
- [ ] Для регулярной услуги выбрать "Вид поступления" — финансовую категорию будущих начислений и платежей. После выбора система оставит только совместимые тарифы.
- [ ] Проверить показанный "Способ расчёта": он определяется выбранным тарифом и не вводится отдельно. В поле "Тариф" одновременно видны название и ставка, которая будет применена при начислении.
- [ ] Указать стоимость регулярной услуги и выбрать `Начисление поставщику`. При сохранении система одним действием создаст отдельный тариф с этой ставкой, свяжет его с услугой и зафиксирует допустимый вид начисления поставщику; введённая сумма сразу появится в основной строке и не обнулится. Кнопка `Добавить услугу` в разделе поставщиков использует тот же сценарий и сразу добавляет полностью настроенную услугу в список выбора поставщика.
- [ ] Проверить вычисленную единицу измерения: `Фиксированно` — `руб.`, `По людям` — `чел.`, `По счетчику воды` — `м³`, `По счетчику электричества` — `кВт·ч`. Единица меняется вместе с тарифом и вручную не редактируется.
- [ ] Выбрать периодичность регулярной услуги: `Ежемесячно` создаёт начисление каждый месяц, `Ежегодно` — один раз в выбранном месяце начисления.
- [ ] Для ежемесячной услуги указать только число оплаты: срок наступит в следующем месяце после начисления. Для ежегодной услуги дополнительно выбрать календарный месяц оплаты; срок относится к тому же учетному году, поэтому поздно сформированное начисление может сразу оказаться просроченным.
- [ ] Чтобы изменить действующую регулярную услугу, нажать кнопку с карандашом в её основной строке. В одной форме можно исправить название, вид поступления, тариф, расписание, срок оплаты, перенос долга и признаки счётчика/порогов; единица измерения обновится по выбранному тарифу. Тип регулярной услуги после создания не меняется; при ошибке сохранения форма остаётся открытой и сохраняет введённые значения для повтора.
- [ ] Проверить признаки "Пороговая тарификация" и "По счётчику" в основной строке каждой услуги. В строках периодичности, срока оплаты, переноса долга и отдельных тарифных порогов эти признаки намеренно не повторяются.
- [ ] При включении "Пороговой тарификации" в форме услуги сразу сверить появившийся список порогов выбранного тарифа: название, верхнюю границу и цену за единицу. Эти значения показываются справочно; изменять состав порогов нужно в строках самого тарифа.
- [ ] Создать тариф: название, вид поступления, база расчета, ставка и дата начала действия.
- [ ] Для электроэнергии нажать `Добавить порог`, указать название, верхнюю границу и ставку. Новая ступень сохраняется в тарифе и участвует в начислениях; для удаления добавленного порога укажите причину, которая попадёт в историю изменений.
- [ ] Использовать базы расчета: fixed, people, meter_water, meter_electricity.
- [ ] При изменении тарифа уже созданные начисления не пересчитываются задним числом.
- [ ] Для нового месяца проверять дату начала действия тарифа до генерации начислений.
- [ ] При объявлении сбора указать взнос и выбрать режим `Все гаражи` либо конкретных участников. Поле `Сумма сбора` заполняется автоматически по формуле `взнос × количество участников`; вручную его изменять не нужно. Для режима `Все гаражи` система использует полное количество активных гаражей, даже если рабочий список загружается частями. Сохраненный сбор сразу появляется в списке `Объявленные сборы` и не исчезает, если одновременно завершилась фоновая загрузка списка. До первого начисления состав можно уточнять.
- [ ] Все одновременно действующие сборы начисляются автоматически за текущий рабочий месяц вместе с регулярными услугами: общий сбор применяется ко всем активным гаражам, выборочный — только к указанным участникам. Сбор, объявленный в середине месяца, относится к этому же учетному месяцу. Начисления сразу видны в форме поступлений, ранее внесённый аванс по прочим доходам уменьшает долг, а повторный автоматический запуск не создаёт дубли.
- [ ] Кнопку `Доначислить` использовать только для ручного дозаполнения выбранного месяца, например после уточнения участников: система пропустит гаражи, которым этот сбор уже начислен. Комментарий необязателен, но при ручной корректировке в нем следует указать причину; действие попадет в историю изменений.
- [ ] После создания начислений не менять исторический состав сбора: система отклонит замену участников, но позволит исправить остальные реквизиты сбора.

## 6. Платежи и начисления

Раздел "Платежи" содержит поступления, выплаты, начисления, начисления поставщикам и показания счетчиков.

- [ ] На вкладке `Поступления` сначала найти один или несколько гаражей через поиск; без выбранного гаража ведомость поступлений не загружается. На вкладке `Выплаты` поиск и карточки гаражей не показываются: форма поставщиков и сотрудников открывается сразу и не зависит от выбора гаража.
- [ ] Работать с шапкой формы сверху вниз: сначала поиск и список выбранных гаражей, затем вкладки, после них единая карточка активного гаража со сведениями, показателями задолженности, расшифровкой и действиями. До выбора гаража его сведения над строкой поиска не показываются.
- [ ] Для одновременной работы с несколькими гаражами можно выполнять несколько поисковых запросов подряд: уже выбранные карточки сохраняются над ведомостью, пока пользователь не уберёт их отдельно или не нажмёт `Очистить`.
- [ ] Результаты поиска гаражей идут по полезности: сначала точный номер, затем номера, начинающиеся с запроса, потом остальные совпадения номера и ФИО; внутри каждой группы применяется естественный порядок номеров.
- [ ] При каждом новом открытии раздела заново выбрать нужные гаражи: строка поиска, выбранные гаражи, локальная ведомость и история предыдущего запроса не восстанавливаются. Сохранённые в базе платежи и начисления при этом не удаляются.
- [ ] В карточке выбранного гаража различать `Общий долг` и `Аванс`. `Просроченная задолженность` — это часть долга с уже наступившим сроком оплаты; оставшаяся часть общего долга ещё не просрочена. Если аванс и просрочка существуют одновременно по разным услугам, это явно поясняется, а открытая таблица ниже показывает услугу, месяц и остаток просрочки.
- [ ] История прошлых платежей по умолчанию скрыта и не загружается при выборе гаража. Для просмотра, изменения, отмены платежа или работы с квитанцией нажать `История платежей` в карточке гаража; после завершения нажать `Скрыть историю`.
- [ ] После действия `Полная оплата` работать с одной квитанцией на весь созданный пакет: в первой действующей строке пакета доступны печать, отмена печати и печать копии, а остальные строки помечены `В единой квитанции`. Перед подтверждением сверить список услуг, суммы по каждой позиции и общий итог. Отменённая финансовая строка в квитанцию не включается.
- [ ] Если администратор включил общую ведомость, использовать вкладки и серверную пагинацию для просмотра поступлений и выплат по всем данным; поиск гаражей остаётся доступен сверху.
- [ ] Внести поступление владельца: гараж, вид поступления, дата, месяц учета, сумма, документ и комментарий.
- [ ] Внести выплату поставщику: выбрать поставщика, проверить автоматически подставленную услугу и статью расхода, затем выбрать тип `С чеком` или `Без чека`, дату, месяц учета, сумму, документ и комментарий. Статью расхода нельзя выбирать независимо от услуги поставщика.
- [ ] Тип `С чеком` проводится через банк. Тип `Без чека` проводится через кассу и одной операцией увеличивает стоимость и оплаченную сумму на одинаковое значение; отдельное начисление и кнопка `Оплатить` для такой строки не нужны. При изменении, отмене или восстановлении выплаты связанная стоимость обновляется синхронно.
- [ ] Обычная выплата поставщику или сотруднику проверяется по доступному остатку банковского счета. Отрицательная `Разница` по отдельной услуге выплату не блокирует и остается видна для сверки недостающих сборов.
- [ ] Оклад активного сотрудника начисляется автоматически каждый месяц. Для дополнительного начисления использовать `Начислить премию` или `Начислить штраф`: выбрать сотрудника и месяц, указать положительную сумму, при необходимости номер документа и обязательно заполнить основание. Премия увеличивает, а штраф уменьшает начисление за выбранный месяц; уже выплаченную сумму штрафом уменьшить нельзя.
- [ ] После премии или штрафа проверить строку сотрудника в форме выплат: в колонке услуги показывается расшифровка оклада, премии и штрафа, а в `Стоимость` — их итог. Финансовый отчёт сотрудника использует тот же итог; исполнитель и основание сохраняются в истории изменений.
- [ ] Для фактической сдачи наличных использовать только действие `Сдать кассу в банк`: указать дату, сумму и при необходимости комментарий. Фонд выбирать не требуется — это самостоятельный перевод, который уменьшает кассу и на ту же сумму увеличивает банк, не меняя остаток ни одного фонда. Обычное пополнение, изъятие или перераспределение фонда меняет назначение средств, но не их физическое расположение.
- [ ] В разделе `Фонды` сначала сверить карточку `Общий нераспределенный пул`: членские и целевые взносы, а также поступления `Прочее` входят в одну доступную сумму и не увеличивают остаток фонда автоматически. Для назначения денег нужному фонду использовать `Пополнить`; каждое ручное пополнение уменьшает общий пул, а изъятие возвращает сумму в него. Повторно распределить уже назначенные деньги система не позволит.
- [ ] Для нового назначения нажать `Создать фонд`, указать уникальное название и сохранить. Чтобы изменить название существующего фонда, открыть действие `Карточка` в его строке; остаток, операции и связи фонда сохраняются по постоянному идентификатору.
- [ ] В карточке существующего фонда проверить раздел `Услуги, оплачиваемые из фонда`: здесь справочно перечислены действующие услуги, связанные с фондом через вид поступления. Изменить привязку из карточки фонда нельзя — для этого используется карточка соответствующей услуги.
- [ ] Чтобы удалить фонд, сначала переназначить все перечисленные в его карточке услуги на виды поступления других действующих фондов. Затем нажать `Удалить фонд`, проверить указанную в подтверждении сумму и ввести причину. Если у фонда есть остаток, система одной операцией вернет его в `Общий нераспределенный пул`, обнулит и удалит фонд; фонд исчезнет из рабочего списка, но автоматическое изъятие, прежние операции и запись в `Истории изменений` сохранятся.
- [ ] В таблице `Ручные перераспределения` проверять ручные пополнения, изъятия, отмены и восстановления. Автоматические назначения поступлений в эту рабочую историю не входят и остаются доступны в общем аудите и отчетах.
- [ ] Если поступление уже израсходовано или сдано в банк, сначала отменить связанные операции; система не позволит отменой или уменьшением поступления сделать остаток кассы несогласованным.
- [ ] При открытии вкладки `Выплаты` проверить текущий месяц; для другого периода выбрать любой месяц и год в локализованном поле `Месяц`. Календарь открывается от левого края поля и полностью остаётся внутри формы, включая первый столбец месяцев и действие `Очистить`, даже на узком экране.
- [ ] В прошлом месяце форма выплат работает как архив: в таблице остаются поставщик, услуга, входящий и исходящий баланс, стоимость и оплаченная сумма. Оперативные колонки `Собрано`, `Разница`, `Действие` и кнопка оплаты в строке не показываются. В текущем и будущем месяце они доступны как обычно.
- [ ] В колонке `Собрано` сверять все доступные средства по услуге: поступления текущего месяца плюс неиспользованный остаток прошлых месяцев. `Разница` показывает остаток после проведённых выплат. Положительная сумма автоматически остаётся доступной в следующем месяце, пока не будет израсходована; отдельную операцию переноса создавать не нужно.
- [ ] В форме выплат сверить `Входящий баланс` и `Исходящий баланс` по каждому получателю и статье расхода. Задолженность показывается отрицательной суммой, аванс — положительной, отсутствие остатка — нулём; эти учетные колонки отображаются нейтральным цветом.
- [ ] В текущем или будущем месяце проверить цвет только в колонке `Собрано`: сумма меньше стоимости услуг за месяц должна быть красной, равная стоимости или превышающая её — зелёной. Остальные денежные колонки и нижние итоги кассы цветом не выделяются.
- [ ] Не создавать ручную операцию переноса между месяцами: исходящий долг или аванс автоматически становится входящим остатком следующего месяца для той же пары получатель + статья расхода.
- [ ] Создать ручное начисление по гаражу с обязательным комментарием.
- [ ] Для штрафа использовать отдельное действие `Начислить штраф` в карточке гаража: указать произвольную положительную сумму, месяц и обязательную причину. Система использует системный вид поступления `Штраф`, добавляет начисление в ведомость и сохраняет исполнителя и причину в истории изменений. Без права `payments.write` действие недоступно.
- [ ] Создать начисление поставщику для сверки обязательств и выплат. Сначала выбрать поставщика: поле `Услуга` заполнится из его карточки автоматически и останется недоступным для произвольного изменения. Если услуга не настроена, открыть карточку услуги, выбрать вид начисления поставщику, затем закрепить эту услугу за поставщиком.
- [ ] Не запускать регулярные начисления вручную: система автоматически обрабатывает активные регулярные услуги и объявленные сборы. Действие `Добавить начисление` использовать только для разового начисления по гаражу.
- [ ] Внести показания воды или электричества по гаражу. В годовой таблице ячейки будущих месяцев отключены, а подробная форма также не позволит создать такое показание.
- [ ] При сохранении показания за текущий месяц система сразу применяет связанный действующий тариф: для воды умножает расход на ставку, для электроэнергии рассчитывает сумму по настроенным порогам, создаёт отсутствующее начисление и показывает его в форме поступлений. Если по этому виду поступления уже есть аванс, он сразу распределяется на новое начисление. Исторические показания не пересчитываются по текущей настройке автоматически.
- [ ] Проверять предупреждения о разрыве истории счетчиков.
- [ ] Обычным способом изменять существующее показание только за текущий учетный месяц. Для прошлого месяца использовать окно `Историческая корректировка`, указать обязательную причину и иметь отдельное право `Исторические показания`; создавать или изменять показание будущего месяца нельзя.
- [ ] После изменения показания проверить автоматически пересчитанное связанное начисление. Если оно уже полностью или частично оплачено, система оставит показание и сумму без изменений и сообщит о конфликте; сначала исправьте оплату штатной операцией или согласуйте отдельную корректировку.
- [ ] Отменять ошибочные операции только через действие отмены с обязательной причиной.
- [ ] Отмененные операции исключаются из активных списков, сводки и расчетов задолженности.
- [ ] При изменении платежа, начисления, выплаты или показания проверить окно `было -> стало`; пустое новое значение может сохраниться сразу, а изменение существующего значения требует подтверждения.
- [ ] После отмены или восстановления операции проверить раздел "История изменений" по гаражу, контрагенту, месяцу или документу.

## 7. Импорт Access

Раздел "Импорт Access" используется для безопасной проверки старой базы.

- [ ] Перед импортом сделать backup PostgreSQL.
- [ ] Выбрать `.accdb` или `.mdb`.
- [ ] Запустить dry-run.
- [ ] Проверить статусы checks: пройдено, предупреждение, ошибка.
- [ ] Если в checks есть "Повторная проверка файла", сверить, что это осознанная повторная загрузка той же Access-БД, а не случайный дубль.
- [ ] Проверить блок "Reader Access": если он показывает "Не настроен", фактическое чтение `.accdb/.mdb` еще не подключено.
- [ ] Проверить лог запуска: шаг получения файла, расчет hash, проверки `check_*` и финальный итог `dry_run_finished`.
- [ ] После фактического переноса проверить вкладку "Создано": там должны быть видны записи, созданные выбранным запуском Access, и их статус rollback.
- [ ] Скачать отчет кнопкой "Скачать отчет JSON".
- [ ] После успешного dry-run нажать "Запросить импорт", указать причину и подтвердить, что backup PostgreSQL создан.
- [ ] Помнить, что до подключения reader Access заявка только фиксируется в истории и не переносит строки в рабочую базу.
- [ ] Если заявка на импорт отправлена ошибочно, нажать "Отменить заявку" и указать причину; это не rollback данных.
- [ ] Не менять исходную Access-БД вручную.
- [ ] Не переносить данные вручную, если dry-run показывает ошибки.
- [ ] Если есть предупреждение про ACE-драйвер или Microsoft Access, передать разработчику JSON-отчет без исходной базы.

## 8. Отчеты

Раздел "Отчеты" содержит восемь рабочих вкладок: `Консолидированный`, `По гаражам`, `По выплатам`, `Поступления`, `Оплаты из кассы`, `Сдача кассы в банк`, `Сборы` и `Изменение фондов`.

- [ ] Консолидированный отчет: выбрать месяц начала и окончания. Для каждого месяца сверить параллельные списки поступлений и выплат по наименованиям, итоговую разницу и остаток банковского счета на начало и конец месяца. Сдача кассы в банк увеличивает остаток счета, банковская выплата уменьшает, а выплата без чека из кассы на него не влияет.
- [ ] `По гаражам`: выбрать период, гаражи, владельцев, виды поступлений и нужную детализацию.
- [ ] Отчет по поступлениям: выбрать даты, вид поступления, гаражи, владельцев и режим строк.
- [ ] Отчет по выплатам: выбрать даты, поставщиков и/или сотрудников, статьи расходов и режим строк.
- [ ] При медленном открытии финансового отчёта дождаться заполнения каркаса итогов и таблицы. Календарь периода доступен уже во время загрузки и должен раскрываться полностью, без обрезания границей окна.
- [ ] Пользователь с правом `audit.read` может перейти из финансового отчёта контрагента в отфильтрованную историю изменений. Без этого права раздел перехода полностью скрыт и не занимает место в отчёте.
- [ ] При сверке поставщика проверить карточку `Входящий остаток`: она учитывает стартовый баланс и все активные начисления и выплаты до начала выбранного периода. В строке входящего остатка колонки `Начислено` и `Оплачено` остаются пустыми, поэтому итоги периода содержат только операции выбранных месяцев; отрицательное значение учитывается как аванс.
- [ ] `Оплаты из кассы` и `Сдача кассы в банк`: сверить движение наличных и поступление той же суммы на банковский счет.
- [ ] `Сборы`: сверить начислено, оплачено, остаток и список гаражей по выбранному сбору.
- [ ] `Изменение фондов`: сверить фонд, вид изменения, сумму до/после, пользователя и комментарий.
- [ ] Режим строк `all` показывает начисления и оплаты/выплаты.
- [ ] Режим `accruals` показывает только начисления.
- [ ] Режим `payments` показывает только оплаты или выплаты.
- [ ] В каждой вкладке использовать `Скачать XLSX` или `Скачать PDF`: выгрузка применяет тот же период, фильтры и сортировку, что экран.
- [ ] Для отчёта `По гаражам` сравнить экран с XLSX/PDF: порядок колонок, строки, месяц `мм.гггг`, итоговые суммы и пояснение должны совпадать; лишней колонки владельца быть не должно.
- [ ] При смене страницы или сортировки сверять общие итоги: они относятся ко всей отфильтрованной выборке, а не только к видимым строкам.
- [ ] Если отчет не сходится, проверить период, фильтры, стартовые балансы, регулярные начисления и показания счетчиков.

## 9. История изменений

Раздел "История изменений" доступен пользователям с `audit.read`.

- [ ] Смотреть последние действия пользователей и системные события.
- [ ] Искать по действию, сущности или описанию.
- [ ] Использовать фильтры раздела, действия, объекта, пользователя, периода и быстрые категории "Только удаления", "Только восстановления", "Только финансы".
- [ ] Открывать карточку события кнопкой "Открыть", чтобы увидеть `было`, `стало`, причину или комментарий и связанные данные.
- [ ] Проверять связанный гараж, контрагента, расчетный месяц или документ, если нужно понять, какая запись повлияла на расчеты.
- [ ] Для удаления, архивирования, восстановления, отмены и ручных корректировок обязательно читать причину: она объясняет, почему действие было выполнено.
- [ ] Проверять события входа, смены пароля, создания пользователей, платежей, импорта и отмен.
- [ ] Скачать CSV через кнопку "Скачать CSV".
- [ ] Не использовать историю изменений как источник полного финансового отчета: для этого есть раздел "Отчеты".

## 10. Что нового

Раздел "Что нового" показывает историю релизов из `GET /api/app-releases`.

- [ ] После обновления открыть раздел и проверить новую версию.
- [ ] Прочитать описание, если изменились правила работы.
- [ ] Если раздел не загрузился, сообщить администратору и проверить troubleshooting.

## 11. Закрытие месяца

Закрытие месяца в GarageBalance означает сверку и сохранение контрольных отчетов. Оно не блокирует период и не создает отдельную финансовую операцию.

- [ ] Выполнить порядок из `docs/monthly-cycle-checklist.md`: backup, справочники, тарифы, счетчики, начисления, поступления, выплаты, касса, банк и фонды.
- [ ] Проверить, что поступления закрывают старейшие начисления того же гаража и вида поступления. В форме платежей значение `Оплачено` по строке не должно превышать `К оплате`, а лишняя сумма должна быть показана отдельно в колонке `Аванс` и в показателе `Аванс на конец`.
- [ ] Создать следующее начисление того же вида и убедиться, что сохранённый аванс применился автоматически, уменьшил долг и не был повторно записан как новая оплата.
- [ ] После частичной оплаты системной услуги, например `Вода`, внести оставшийся долг в той же строке. Платёж должен сохраниться по указанному в строке виду поступления без открытия справочников и без сообщения `Не найден вид поступления`.
- [ ] В истории платежей открыть платёж, изменить только комментарий и один раз нажать `Сохранить`. Изменение должно записаться без второго подтверждения; при корректировке суммы, даты, месяца или документа подтверждение должно сохраниться.
- [ ] Проверить карточку гаража с задолженностью: отдельной кнопки `Перенести задолженность` быть не должно. Долг учитывается по сохранённым начислениям и платежам без ручной операции между месяцами.
- [ ] Убедиться, что частично оплаченный годовой взнос продолжает показываться до полной оплаты.
- [ ] Сверить следующий месяц: долг или аванс поставщика переносится только внутри той же пары `поставщик + статья расхода`; ручную строку переноса создавать не нужно.
- [ ] Сверить остаток собранных средств по услуге: если в июне собрано `9 243,81` и выплаты не было, в июле эта сумма должна войти в `Собрано`; последующая выплата уменьшает остаток, а не обнуляет его произвольно.
- [ ] Проверить все восемь вкладок отчетов по одному периоду и сохранить нужные XLSX/PDF вне Git.
- [ ] Записать спорные строки и контрольные суммы. Если старый месяц исправляется позже, использовать штатное изменение, отмену или восстановление с причиной и затем повторить сверку последующих месяцев.

## 12. Ошибки и проверка форм

- [ ] Если форма показывает validation summary, исправить все перечисленные поля.
- [ ] Если API вернул `unauthorized`, нужно войти заново.
- [ ] Если API вернул `forbidden`, у пользователя нет нужного права.
- [ ] Если API вернул `validation_failed`, проверить поля формы.
- [ ] Если ошибка повторяется, записать время, раздел, действие и безопасный текст ошибки.
- [ ] Не отправлять разработчику пароли, токены, `.env`, Access-БД, backup и полные финансовые таблицы.
