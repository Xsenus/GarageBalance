# Stage 8 Feedback Template

Этот шаблон используется после ручной демонстрации этапа 1. Он нужен, чтобы замечания заказчика не потерялись и могли быть перенесены в `docs/project-roadmap.md` как отдельные проверяемые задачи.

## Итоги Показа

- Дата демонстрации:
- Стенд или база:
- Коммит или версия:
- Участники:
- Роли, которые показали:
- Разделы, которые прошли:
- Отчеты и выгрузки, которые открыли:

## Решение По Демонстрации

Выбрать один вариант:

- [ ] Замечаний нет, этап можно считать принятым после согласования.
- [ ] Есть замечания, их нужно внести в roadmap.
- [ ] Нужна повторная демонстрация после исправлений.

## Правила Фиксации

- Не переносить в Git реальные паспортные данные, полные адреса, телефоны, банковские реквизиты, платежные документы, raw Access rows, `.accdb`, `.mdb`, `.pgdump` или `.env`.
- Если замечание зависит от реальных данных, описать только обезличенный сценарий и ожидаемое поведение.
- Каждое подтвержденное замечание переносить в roadmap отдельным checkable item с понятным статусом.
- Если замечаний нет, добавить в `История выполнения` дату, участников и подтверждение отсутствия мотивированных замечаний.
- Пункт `Собрать замечания и внести их в roadmap` можно закрыть как `[x]` только после заполненного списка замечаний или явного подтверждения, что замечаний нет.
