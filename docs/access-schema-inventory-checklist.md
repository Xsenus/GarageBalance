# Access Schema Inventory Checklist

Этот checklist описывает безопасное снятие полной схемы старой Access-БД перед фактическим импортом.

## Предусловия

- [ ] Есть приватная рабочая копия `.accdb`/`.mdb`; оригинал не изменяется.
- [ ] Подтвержден рабочий reader: ACE OLE DB/ODBC provider, Microsoft Access runtime или согласованная конвертация.
- [ ] Проверка выполняется вне Git и без выгрузки персональных, финансовых и raw import rows в репозиторий.

## Что нужно снять

- [ ] Список пользовательских таблиц.
- [ ] Количество строк по каждой пользовательской таблице.
- [ ] Список колонок: имя, тип, nullable, размер/precision/scale, default value.
- [ ] Primary keys и уникальные индексы.
- [ ] Foreign keys/relationships, включая nullable-связи.
- [ ] Indexes, которые влияют на поиск, отчеты, импорт и дедупликацию.
- [ ] Системные или временные таблицы, которые нельзя импортировать как бизнес-данные.
- [ ] Список queries/forms/reports только как технические имена без содержимого персональных строк.

## Минимальный безопасный результат

Итоговый inventory-файл или roadmap history может содержать:

- имя приватной копии без персональных деталей;
- SHA-256 и размер файла;
- provider/способ чтения;
- список таблиц, колонок, связей и row counts;
- список обнаруженных рисков импорта.

Итог не должен содержать:

- ФИО, адреса, телефоны, паспортные данные;
- платежные документы и суммы по конкретным владельцам;
- raw rows из Access;
- сам `.accdb/.mdb` файл, дампы или screenshots с персональными/финансовыми данными.

## Пример технической фиксации

```text
provider=Microsoft.ACE.OLEDB.16.0
copyName=gsk-access-working-YYYYMMDD.accdb
copySha256=<hash>
tables=42
relationships=37
tablesWithRowCounts=42
rawRowsExported=false
```

## Когда roadmap-пункт можно закрыть

- [ ] Таблицы, колонки, ключи, связи, индексы и row counts сняты с приватной копии.
- [ ] Результат сверки сохранен без персональных/финансовых строк.
- [ ] Privacy-check подтверждает, что Access-файлы и raw exports не попадут в Git.
- [ ] Следующие пункты roadmap по контрольным суммам и карте Access -> PostgreSQL могут использовать этот inventory.
