# Проверка Производительности Отчетов Stage 7

Этот документ фиксирует состояние пункта Stage 7 `Проверить производительность отчетов на периодах в несколько месяцев и на полном объеме базы`.

## Что уже закреплено автоматически

- Экранный консолидированный отчет, отчет по поступлениям и отчет по выплатам без текстового поиска не должны материализовать весь набор строк ради первых видимых строк.
- `BackendPerformanceGuardTests.ScreenReportQueries_UseDatabaseLimitsForVisibleRows` проверяет, что `ReportService` сохраняет серверные ограничения видимых строк через `ApplyReportRowLimit`, `Take(NormalizeReportLimit(...))`, `CountAsync` и `SumAsync`.
- Итоги отчетов считаются отдельными database aggregate-запросами, а `rowCount` берется через `CountAsync`, чтобы экран мог показывать количество строк без загрузки полной выборки в память приложения.
- Общий финальный чеклист `docs/final-performance-checklist.md` уже описывает `EXPLAIN (ANALYZE, BUFFERS)`, индексы, bundle budget, browser console и acceptance thresholds для полного performance gate.

## Почему пункт не закрыт как `[x]`

Полное закрытие требует живого PostgreSQL-прогона на реалистичном объеме данных ГСК:

- периоды отчетов в несколько месяцев;
- полная база или близкая к ней тестовая копия;
- проверка response time и памяти приложения;
- проверка browser console и отзывчивости таблиц;
- при заметной задержке - `EXPLAIN (ANALYZE, BUFFERS)` для самых тяжелых запросов.

На текущей машине этот live-прогон невозможен: `postgresTcp=False`, `psql=False`, `docker=False`, реалистичная копия данных отсутствует. Поэтому пункт переведен в `[acceptance]`: кодовые guard-ы выполнены, но приемка на реальной PostgreSQL-БД остается открытой.

## Что нужно сделать для закрытия

- Поднять локальную или VPS PostgreSQL-БД с реалистичными данными.
- Применить актуальные EF migrations.
- Открыть отчеты за один месяц и за несколько месяцев: консолидированный, поступления, выплаты, касса, банк, сборы и фонды.
- Зафиксировать фактические timings, отсутствие ошибок в backend/frontend logs и browser console.
- При медленных запросах приложить вывод `EXPLAIN (ANALYZE, BUFFERS)` без персональных, платежных или импортных данных.
- После успешной приемки обновить этот документ и строку roadmap с `[acceptance]` на `[x]`.
