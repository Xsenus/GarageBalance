# Meter Reading Operator Scenario Verification

Этот документ фиксирует evidence для закрытия верхнеуровневого сценария оператора по показаниям счетчиков.

## Закрытая Область

- [x] Оператор вводит и изменяет показания воды и электроэнергии через backend API с audit-историей.
- [x] Расход рассчитывается от предыдущего активного или стартового показания гаража.
- [x] Годовая таблица загружает сохраненные показания и записывает новые значения через API.
- [x] Пропущенные показания и разрывы последовательности видны оператору.
- [x] Регулярные начисления поддерживают тарифы `meter_water` и `meter_electricity`.
- [x] Месячная ведомость гаража показывает показание, расход, начисление, оплату и задолженность.
- [x] Сводный отчет показывает количество показаний по месяцам и гаражам.

## Проверочное Покрытие

- [x] `FinanceServiceTests` проверяет расчет расхода, округление, разрыв истории, дубли, отмену, восстановление и начисления по тарифам счетчиков.
- [x] `FinanceServiceTests.GetGarageIncomeWorksheetAsync_BuildsRowsFromAccrualsPaymentsAndMeters` проверяет месячную ведомость с показанием, расходом, начислением, оплатой и долгом.
- [x] `ReportServiceTests.GetConsolidatedReportAsync_ReturnsMonthlyTotalsAndGarageRows` проверяет счетчики показаний в месячных и гаражных строках отчета.
- [x] `App.test.tsx` проверяет годовую таблицу, сохранение показаний, рассчитанный расход и подсветку гаражей без показаний.
- [x] `ProjectWideRoadmapStatusTests.MeterReadingOperatorScenarioIsCompleteWhenYearTableWorksheetAccrualsDebtAndReportsAreCovered` удерживает roadmap-status вместе с evidence.
