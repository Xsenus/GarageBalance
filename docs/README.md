## Правила актуальности

1. Корневой `README.md` описывает только начало работы, запуск, сборку и навигацию.
2. Подробности механизма находятся в одном тематическом документе, без копий в других файлах.
3. Пользовательские изменения описываются в `backend/GarageBalance.Api/AppReleases/releases.json`.
4. Временные результаты аудита, тестовые логи и отчёты не коммитятся как документация.
5. Все относительные Markdown-ссылки проверяются автоматическим тестом.
