# Интеграции GarageBalance

## Импорт Microsoft Access

Импорт используется для переноса старой базы `.accdb`/`.mdb` без ручного повторного ввода.

Безопасный процесс:

1. создать backup PostgreSQL;
2. загрузить рабочую копию Access-файла;
3. выполнить dry-run;
4. проверить найденные сущности, предупреждения и ошибки;
5. разобрать quarantine;
6. подтвердить применение импорта;
7. сверить количества, суммы и показания;
8. сохранить технический отчёт без исходных персональных данных.

Повторный импорт защищён fingerprints строк. Фактическое применение и rollback выполняются как наблюдаемые операции и фиксируются в audit. Raw-файлы и снимки строк не публикуются в Git и не возвращаются обычным API.

## 1C Fresh

Интеграция строится как backend-синхронизация: токены хранятся через ASP.NET Core Data Protection, браузер не получает plaintext. Перед фактическим обменом выполняется preview, после чего пользователь подтверждает операцию.

Синхронизация должна быть идемпотентной, иметь внешний идентификатор, журнал запуска, повтор после временной ошибки и понятную сверку созданных/обновлённых/пропущенных записей.

Набор передаваемых данных и рабочий контур согласуются отдельно до включения production-синхронизации.

## Печать квитанций и чеков

Печать отделена от финансовой операции: сначала backend сохраняет и подтверждает платёж, затем создаётся задание печати. Ошибка устройства не должна создавать второй платёж.

Действие `Полная оплата` присваивает всем созданным строкам одного гаража и одной даты общий идентификатор пакета. При печати backend собирает действующие поступления пакета, их виды услуг и активные распределения по начислениям, вычисляет единый итог и передаёт адаптеру одно задание с позициями. Обычное одиночное поступление без пакета сохраняет прежний сценарий одной квитанции. Пакет с выплатой, другим гаражом или другой датой отклоняется; один документ ограничен 100 финансовыми позициями.

Шаблон документа получает только необходимые реквизиты. Секреты оборудования и интеграционные адреса хранятся как защищённые настройки. Каждая попытка печати имеет статус и запись в истории.

До подключения реального оборудования необходимо подтвердить модель устройства, обязательные реквизиты, формат документа и юридические требования.
