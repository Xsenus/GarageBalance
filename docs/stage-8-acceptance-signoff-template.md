# Stage 8 Acceptance Signoff Template

Этот шаблон используется только после ручной демонстрации и разбора замечаний по этапу 1. Он нужен, чтобы зафиксировать итоговое решение: этап принят, принят с уже внесенными замечаниями, не принят из-за мотивированных замечаний или требует повторной демонстрации.

- Дата решения:

  - `docs/stage-8-demo-script.md`
  - `docs/stage-8-feedback-template.md`

- Проверки перед решением:
  - backend tests:
  - frontend tests:
  - build/publish:
  - миграции или причина, почему локальная PostgreSQL-БД недоступна:

## Решение По Приемке

Выбрать один вариант:

- [ ] Принято без мотивированных замечаний.
- [ ] Принято, замечания уже внесены в roadmap как отдельные задачи.
- [ ] Не принято: мотивированные замечания перечислены ниже и должны быть внесены в roadmap.
- [ ] Нужна повторная демонстрация после исправлений.

## Правила Фиксации

- Не переносить в Git реальные паспортные данные, полные адреса, телефоны, банковские реквизиты, платежные документы, raw Access rows, `.accdb`, `.mdb`, `.pgdump` или `.env`.
- Если замечание основано на реальных данных, фиксировать только обезличенный сценарий, ожидаемое поведение и ссылку на roadmap-пункт.
- Если выбран вариант "Принято без мотивированных замечаний", добавить в `История выполнения` дату, участников и явное подтверждение отсутствия мотивированных замечаний.
- Если замечания есть, сначала перенести их в roadmap отдельными checkable items, затем ссылаться на эти строки в этом шаблоне.
- Пункт `Получить приемку этапа 1 или список мотивированных замечаний` можно закрыть как `[x]` только после заполненного решения по приемке или явного подтверждения пользователя/заказчика.
